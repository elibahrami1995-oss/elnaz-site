## ⚠️ نکته‌ی فنی حیاتی — حتماً اول بخون

سایت روی **Next.js** ساخته شده، نه HTML خام. محتوای واقعی صفحه از فایل‌های زیر میاد:

```
app/page.tsx                          ← ترتیب کلی بخش‌ها
app/components/HeroSection.tsx        ← بخش هرو
app/components/AboutSection.tsx       ← بخش معرفی
app/components/JourneySection.tsx     ← تایم‌لاین
app/components/BrandsSection.tsx      ← پکیج‌ها + CTA میانی
app/components/EducationSection.tsx   ← آموزش/کلاس + پرامپت رایگان
app/components/PortfolioSection.tsx   ← نمونه‌کار
app/components/ContactSection.tsx     ← فرم تماس
app/components/Header.tsx             ← navbar
app/globals.css                       ← تمام استایل‌های مشترک (رنگ، دکمه، فاصله‌گذاری)
```

**یک فایل `index.html` در ریشه‌ی پروژه وجود دارد که اصلاً استفاده نمی‌شود** — این فایل یک بازمانده‌ی قدیمی است. هر تغییری که روی `index.html` داده شود، **هیچ اثری روی سایت واقعی ندارد**. همیشه فقط فایل‌های داخل `app/components/` و `app/globals.css` را ویرایش کن.

### ⚠️ نکته‌ی حیاتی درباره‌ی دیپلوی

**قبل از هر `liara deploy`، حتماً VPN را خاموش کن.** اگه VPN روشن باشه، آپلود قطع می‌شود (خطای `Upload failed`). این چندبار باعث اتلاف وقت شد تا کشف شد.

### روند صحیح هر تغییر

```bash
cd ~/Desktop/elnaz-site
# ... ویرایش فایل‌های app/components/*.tsx یا app/globals.css ...
npm run dev              # تست لوکال روی http://localhost:3000
git add app/ public/
git commit -m "توضیح تغییر"
git push
liara deploy              # حتماً با VPN خاموش
```

چارچوب پیام‌رسانی بر اساس کتاب *StoryBrand* (SB7) طراحی شده — دو BrandScript کامل (برای هر دو مخاطب) ساخته شده که مبنای تمام کپی‌رایتینگ سایت است.

## فاز ۳ — بک‌اند (هنوز شروع نشده، طبق برنامه بعد از تکمیل کامل UX/UI)

برنامه‌ی آینده:
- دیتابیس: PostgreSQL روی Liara (نه Supabase)

- در آینده‌ی دورتر: احراز هویت کاربر + پرداخت با ZarinPal برای دوره‌های پولی

تخمین زمان اولیه (فقط ذخیره‌ی فرم‌ها، بدون auth/پرداخت): حدود ۳ تا ۵ جلسه‌ی کاری مشابه جلساتی که تا الان داشتیم.
